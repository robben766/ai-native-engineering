# 平台级经验（主线通用）— 模板

> 复制为目标项目的 `docs/experience/platform.md`。
> 这一页记录"所有仓库都要检查的通用坑"，供 review 和用例生成时逐条比对。
> 下面按维度给出"该写哪些内容"的通用提示；把 `____` 换成你项目的真实约定，删掉不适用的小节。
> 铁律：只写结构性/架构级的通用坑，每条尽量带一处证据（代码位置 / 工作项编号），便于后续人复核。

---

## 怎么用这张模板

- 每个小节回答两个问题：**这类坑长什么样（识别特征）** + **QA 该怎么查（必查项 / 测试点）**。
- 不确定某维度是否适用就先留占位，等第一次踩坑后再补实例——经验层是"越用越厚"的。
- 教学性示例请用假数据，不要把真实客户名、真实缺陷编号、真实逃逸率写进这里（那些属于内部落地层，不进对外模板）。

---

## 1. 多租户隔离（通常是最高频问题）

写这一节前先确认你的隔离模型，再照着填：

- **租户识别方式**：`请求头 / 子域名 / token claim / ____`；注意是否存在**新旧两套字段名并存**（旧代码用老字段，容易漏）。
- **隔离级别**：`独立库 / 独立 schema / 行级 tenant_id / ____`。
- **必查**：
  - 新增/修改的查询、列表、导出接口是否**正确带上租户上下文**？漏了会**跨租户串数据**（严重）。
  - 定时任务、异步任务、消息消费里是否丢失租户上下文？
  - 缓存 key 是否按租户隔离？否则 A 租户读到 B 租户缓存。
- **测试点**：至少用 2 个租户各跑一遍，验证数据不串；切换租户后列表/详情正确。

## 2. 核心计算引擎（禁止绕过）

若项目有统一的「核心计算引擎」承载金额/费率/规则类计算，务必写清"什么必须走引擎、什么算绕过"：

- **必须走引擎的数值**：`金额 / 费率 / 分成 / 结算 / ____`——严禁手工或硬编码计算。
- **调用/排查链**：`<编排模块> → <核心计算引擎> → <数据模块>`（按你项目实际填）。
- **必查**：review 里若看到手写的金额/费率计算，是风险点；用例里要对核心数值做**与引擎结果一致性**验证。

## 3. 权限模型

- **权限数据维护在哪**：`____`（哪个模块/仓、用什么机制下发）。
- **必查**：新接口是否配了权限点？不同角色（管理员/普通/只读）访问是否符合预期？越权访问是否被拦截？
- **测试点**：用不同角色账号各验证一遍可见性与操作权限；重点覆盖"改了数据但没配权限点"的场景。

## 4. 跨仓联动配套（易漏改）

配置类/字段类需求常跨多个仓，**任一漏改都会导致功能不完整**。把你项目典型的联动链写下来：

```
<DB migrate 建表/加字段> → <后端 API> → <管理端前端> → <业务端前端>
```

- **必查**：后端加了字段，前端是否有对应展示/录入？migrate 是否先行？
- 记录**易牵连多仓的高危改动类型**（如涉及金额类/回传类/权限类的 `<业务模块>`），提醒 review 时主动扩散检查面。

## 5. 遗留 / 未迁移业务（不要在别处重复找或改）

如果存在"新老架构并存、部分业务还在旧系统"的情况，列一份清单，避免在错误的仓里白找：

- **仍在旧系统的功能**：`<业务模块>`、`<路由前缀>`、`DB migrate`、权限维护、租户信息同步 等——按实际填。
- 标注每项的正确落点（`<业务仓> / <模块>`），减少判仓歧义。

## 6. 网关与租户路由（如有统一入口）

- 统一入口在哪、负责什么：`<网关模块>`，通常负责租户路由、鉴权、转发。
- **排查提示**：接口异常先排查是否网关层路由/鉴权问题，再往业务层查。

## 7. 通用白盒 review 关注点（跨项目通用清单）

这一节基本可直接复用，是所有语言/框架都适用的检查类目：

- 空值 / 边界 / 异常分支处理
- 分页与大数据量
- 并发与事务边界
- SQL 注入与性能（N+1、缺索引）
- 金额精度（用定点/高精度类型，禁 float/double 直接算钱）
- 时间与时区
- 幂等性（重复提交 / 回调重放）
- 兼容性（新老字段、接口向后兼容）
- 日志与可回滚

## 8. 环境 / 工具坑（把踩过的降级手法记下来）

写"当某工具挂了怎么绕过"这类可复现的应急手法。示例结构（占位，替换成你项目的真实工具与命令）：

- **`<代码平台>` MCP 鉴权失败 → 本地 ref 回退**：当访问令牌未配或缺权限，MCP 取 MR/PR 变更会失败。仓库已本地 clone 时，可直接用本地 git 取 diff（走 SSH，不需 API token）：
  ```bash
  git -C <仓> fetch origin 'refs/merge-requests/<iid>/head:mr-<iid>'
  git -C <仓> diff origin/<主分支>...mr-<iid>
  ```
  提测给的短 SHA 一般就是 MR head 提交，可 `git log --oneline mr-<iid>` 核对。
