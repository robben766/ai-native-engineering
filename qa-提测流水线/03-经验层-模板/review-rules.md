# 代码 Review 规则 + 门禁裁决（模板）

> 这是**空模板**：结构、机制、通用类目和教学示例都在，但所有业务专属条目都替换成了占位符。
> 按你自己的业务，把标「（示例，按你的业务替换）」的行填成真实检查项，并在第五节基线表里给它们定级。

代码 review 门禁 skill 运行时加载。分两部分：**检查清单**（找什么问题）+ **裁决规则**（怎么判放行）。

## 一、检查清单（逐类比对 diff）

| 类别 | 检查要点 |
|------|---------|
| **需求一致性** | diff 是否真正实现了「工作项系统」里的需求/缺陷？有无漏实现、有无超出范围的多改？验收标准逐条能否满足？ |
| **多租户隔离** | 查询/列表/导出/缓存/异步任务是否带租户上下文？（见「领域文档」多租户规则） |
| **核心计算引擎** | 核心金额/费率是否手写计算，而没有走统一的「核心计算引擎」？（见「领域文档」计算规则） |
| **权限** | 新接口是否配权限点？越权是否拦截？ |
| **跨仓配套** | 后端字段是否有前端/DB migrate 配套？（多仓协同变更是否成套） |
| **空值/边界/异常** | NPE、空集合、越界、异常吞没、错误码是否正确返回 |
| **并发/事务** | 事务边界、幂等、重复提交/回调、锁范围 |
| **SQL/性能** | 注入、N+1、缺索引、大数据量分页、全表扫描 |
| **金额精度** | 用定点/高精度类型（如 BigDecimal），禁 float/double；四舍五入策略 |
| **兼容性** | 接口变更是否破坏老客户端？DB 变更是否可回滚？枚举新增是否有默认分支 |
| **日志/可观测** | 关键路径是否有日志；异常是否可追踪租户+业务主键 |
| **批量校验数据源** | 内存查重（stream anyMatch 等）依赖的列表数据源 projection 是否**包含被比较字段**？漏 select 字段会导致该字段恒 null、查重静默失效（教学示例：某查重查询漏 select 关键字段，导致重复数据静默入库） |
| **白名单 vs 黑名单** | 需求给"只对 X/Y/Z 生效"（白名单）时，代码若用"排除 A/B"（黑名单）实现，需确认二者在全部枚举值上等价，别遗漏中间状态 |
| **回传类模块 —— 落库唯一键**（示例，按你的业务替换） | 「<外部系统>」回传落库的唯一键是否含足够的**业务维度**？同键多条的合并/保留规则是否明确并留痕？关键字段解析失败是否**拦截+告警**而非静默入库？（回传类是常见的生产逃逸高发区） |
| **权限类模块 —— 数据权限多路径**（示例，按你的业务替换） | 归属/机构数据权限过滤是否**搜索、列表、导出、异步任务多条路径都覆盖**？归属类字段取数口径是否列表=详情=导出一致？ |
| **金额类模块 —— 金额一致性**（示例，按你的业务替换） | **明细合计=主表金额**、**列表金额=详情金额** 是否成立？费率是否按业务枚举全量正确匹配、不串档？（财务对账类是常见的生产逃逸高发区，判准见 oracles.md） |
| **回调幂等**（示例，按你的业务替换） | 外部回调是否处理"回调成功但落库失败""重复回调""覆盖场景"？管理端状态是否以回调落库为准、可人工补偿？ |
| **批量操作完整性**（示例，按你的业务替换） | 批量导出/更新是否与单条口径一致？导出**行数=筛选数、无整批重复、字段不为空**；批量**可更新字段集=单条可更新字段集**，不静默跳过；导入是否兼容格式变体？批量走的 projection/循环是否漏字段（见"批量校验数据源"）？ |
| **派生金额随源自动重算**（示例，按你的业务替换） | 比例类字段变了，对应**金额**是否自动重算？外部回传带比例时是否触发计算？改一处留旧值＝资损 |
| **多处/端间数据一致性**（示例，按你的业务替换） | 同一数据在多处是否一致——**A 端=B 端**、**系统计算值=外部来源值**、**报表合计=明细**、列表=详情？同步是否丢字段？（是"金额一致性"向端间/多视图的推广） |
| **关系/时点正确性**（示例，按你的业务替换） | 关系链（如推荐/归属关系）变更是否**留痕不丢**？涉及历史的结算是否按**业务时点**取当时关系，而非当前？ |
| **生成/发布幂等**（示例，按你的业务替换） | 会生成下游数据或对外发布的操作，是否幂等——一份来源只生成**一份**结果？同周期报表**不可重复发布**？重复触发是否拦截？ |

## 二、严重度分级

| 级别 | 定义 | 示例 |
|------|------|------|
| **Blocker** | 会导致数据错误/资损/跨租户串数据/核心功能不可用 | 列表漏租户过滤；金额算错；越权 |
| **Major** | 明显功能缺陷或高风险，但非致命 | 边界未处理导致部分场景报错；缺权限点 |
| **Minor** | 不影响功能的质量问题 | 日志缺失、命名、轻微性能 |
| **Info** | 建议/观察，非必须改 | 可读性、重复代码 |

## 三、门禁自动裁决（可覆盖）

**定级不主观**：每个问题的严重度**查第五节「严重度基线表」**确定，不逐次拍脑袋。基线未覆盖的新类型才回退到第二节通用定义临时定级，并记录待补进基线。

```
若 Blocker 数 > 0            → 不通过（拦截，停在门禁，回写「工作项系统」，等修复）
否则若 Major 数 > 0          → 有风险（默认不通过，但提示 QA，可手动覆盖放行）
否则                        → 通过
```

- 裁决结果始终标注 `【自动裁决：通过 / 有风险 / 不通过】`，并列出各级别数量。
- QA 可**手动覆盖**：`覆盖放行` 或 `覆盖拦截`，覆盖时记录理由。

```

```

## 五、严重度基线表（门禁的权威定级表）

> 这是门禁裁决的**权威定级表**——代码 review 门禁给问题定级时**查此表**，不再逐次主观判断。违反标 **Blocker** 的项 → 硬拦，停在门禁。
>
> 下表按"是否导致资损/数据错误/跨租户/核心不可用"定级。后续如需调整某条严重度，改此表即生效；某类问题反复逃逸 → 考虑升级。
>
> **签核机制（占位）**：建议加一列「签核」，由 QA 负责人对每条定级签核后生效。本模板留空，按你团队流程填。

### 通用项（可直接沿用）

| 检查项 | 严重度 | 依据（为什么这个级别） |
|--------|:---:|------|
| 多租户隔离（漏租户过滤） | **Blocker** | 跨租户串数据＝资损级，无争议 |
| 核心计算被手写替代（金额/费率未走「核心计算引擎」） | **Blocker** | 核心金额算错＝直接资损 |
| 权限 / 越权未拦截 | **Blocker** | 越权访问＝数据泄露/篡改 |
| 金额精度用 float/double | **Blocker** | 精度丢失＝对账不平、资损 |
| SQL 注入 | **Blocker** | 安全 |
| 需求一致性 —— 核心需求漏实现 | **Blocker** | 提测的东西没做到 |
| 批量校验数据源漏 select 字段（查重静默失效） | **Blocker** | 数据错误且隐蔽 |
| 并发/事务/幂等 —— **涉及资金或外部回调** | **Blocker** | 重复扣款/重复回调＝资损 |
| 并发/事务/幂等 —— 一般业务 | Major | 非资金场景，风险可控 |
| 空值/边界/异常（部分场景报错、异常吞没非关键路径） | Major | 功能缺陷，非致命 |
| 兼容性（破坏老客户端 / DB 不可回滚 / 枚举缺默认分支） | Major | 破坏线上则升 Blocker，默认 Major |
| 跨仓配套缺失（后端字段无前端/migrate） | Major | 功能不完整，非资损 |
| 白名单/黑名单实现不等价 | Major | 边缘枚举遗漏，需人确认 |
| 需求一致性 —— 超范围多改 | Major | 引入未评审风险 |
| SQL 性能（N+1 / 缺索引 / 全表扫描） | Major | 性能退化，非数据错 |
| 日志/可观测缺失 | Minor | 不影响功能 |

### 业务专属项（示例，按你的业务替换 —— 每类留一行占位）

| 检查项 | 建议严重度 | 依据（为什么这个级别） |
|--------|:---:|------|
| 金额类模块 —— 金额一致性（明细合计≠主表 / 列表≠详情 / 费率串档） | **Blocker** | 财务对账类高发，资损级（示例，替换成你的金额模块） |
| 回传类模块 —— 落库唯一键漏业务维度 / 关键字段解析失败静默入库 | **Blocker** | 回传类高发，数据错误（示例，替换成你的对接方与唯一键规则） |
| 权限类模块 —— 数据权限多路径有一条漏过滤 | **Blocker** | 越权看到他方数据（示例，替换成你的数据权限维度） |
| 回调幂等（重复回调 / 回调成功落库失败 / 覆盖未处理） | **Blocker** | 状态错乱 + 潜在资损（示例，替换成你的回调场景） |
| 批量操作完整性（导出重复/丢失、批量漏更新字段） | **Blocker** | 批量类高发，数据错误（示例，替换成你的批量场景） |
| 派生金额未随源重算（比例变金额没变） | **Blocker** | 派生金额算错＝资损（示例，替换成你的派生字段） |
| 多处/端间数据不一致（A 端≠B 端、系统≠外部来源、合计≠明细） | **Blocker** | 不一致＝资损或合规风险（示例，替换成你的端/视图） |
| 生成/发布不幂等（重复结果、同周期报表重复发布） | **Blocker** | 下游数据重复＝数据错误、合规风险（示例，替换成你的发布场景） |
| 关系链丢失/未按时点匹配 | Major | 影响结算正确性，但可追溯修正（示例，替换成你的关系链） |

**落地方式**：把最终严重度回填到第一节检查清单每行（或让门禁 skill 读本表），门禁即按「违反任一 Blocker 项 → 不通过」确定性裁决。

---

> 自我迭代机制：漏检的问题类型 → 加进"检查清单"；某类问题反复出现 → 考虑升级其默认严重度并回填基线表。
