# AI 原生工程实践 · AI-Native Engineering

用 AI 把工程质量的做法，沉淀成能跨团队、跨公司复制的方法、引擎与模板。

> **在线浏览**：https://robben766.github.io/ai-native-engineering/ &nbsp;·&nbsp; 文档查看器（所有 md 友好渲染）：[qa-提测流水线/查看.html](https://robben766.github.io/ai-native-engineering/qa-提测流水线/查看.html)
>
> 首个模块是 **QA 提测验证流水线**（`qa-提测流水线/`），研发相关模块后续加入。

## 这是什么

工程团队里有很多"每次都要做、却长期靠人、又没沉淀"的固定工作——提测把关、代码评审、回归、质量度量……这套东西把它们用 AI 编排起来，让每次都按同一套标准跑完，并把判断依据沉淀下来，越用越准；再把这套做法整理成能直接搬到别的团队的方法、引擎与模板。

## 怎么讲

每个模块都按同一条线讲清楚：

- **这是什么** —— 解决哪段工作
- **解决什么问题** —— 现状的痛点
- **难点在哪** —— AI 能扎多深、边界在哪，不回避
- **我们的方法** —— 怎么做的
- **怎么落地** —— 照手册搬进你的团队

## 给谁看

| 角色 | 看哪块 |
|------|--------|
| 负责人 / PM | 方法论：AI 在研发全周期能扎多深、价值怎么证。不用懂代码 |
| 研发 | 引擎与规则：变更分析、代码评审、质量门禁、检查清单 |
| QA | 全流程：用例生成、E2E 验证、度量与逃逸率 |
| 想落地的团队 | 落地手册 + 经验层模板 |
